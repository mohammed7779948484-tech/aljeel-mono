## Detected Issues

### Legacy Duplicates

1. **Announcement (6 fields) vs Announcements (7 fields)** — Two DocTypes for the same concept. The `Announcements` version has `display_order` and is likely the newer version. **Decision:** Merge into single `announcements` collection in Phase 2.

2. **Centers (16 fields) vs University Centers (7 fields)** — Overlapping content. `Centers` is more complete. **Decision:** Merge into single `centers` collection in Phase 2.

### Merged Entities

3. **AAU Page + Pages + Static Page** — Three page-like DocTypes with slight variations. All have slug, title_ar/en, content_ar/en, published status. **Decision:** Merge into single `pages` collection with Puck layout support.

### Frappe-Specific Fields Ignored

The following Frappe UI-only layout fields are present in many DocTypes but carry no data:
- Section Break
- Column Break
- Tab Break

These are **not** created as Payload fields.

### Website Settings Cleanup

The `Website Settings` DocType has 93 fields, many Frappe-specific:
- `website_theme`, `navbar_template`, `footer_template` → Frappe-specific, do not migrate
- `brand_html`, `banner_html`, `head_html` → Frappe-specific
- `google_analytics_id`, `robots_txt` → Migrate to `site-settings` or SEO plugin
- Contact page fields → Already in `Contact Page Settings`, remove duplicates
- Social/footer/branding fields → Clean and keep relevant ones

**Decision:** Create a clean `site-settings` global with only the fields needed for the new platform.
